## 📋 專案概述

ARIA v3.0 是一個整合即時雨量監測與避難所風險評估的智能系統，專注於**花蓮縣與宜蘭縣**的災害風險動態評估，能夠回答「**現在哪裡最危險？**」這個關鍵問題。

### 🎯 核心功能
- **🌧️ 即時雨量監測**：整合 CWA API 與歷史颱風資料
- **🏠 避難所風險評估**：結合地形坡度、河川距離、高程等因子
- **🗺️ 動態風險地圖**：即時更新風險分級（URGENT/CRITICAL）
- **📊 災害模擬**：以鳳凰颱風 (2025) 作為壓力測試情境

### 🗺️ 分析邏輯

#### 1. **地形風險評估**
```python
# 地形因子
- 坡度 > SLOPE_THRESHOLD (30°)
- 高程 < ELEVATION_LOW (100m)
- 河川距離分級 (<500m, 500-1000m, >1000m)

# 風險等級
- low: 基礎安全條件
- medium: 距離河川 500~1000m 或高程過低
- high: 距離河川 <500m 或坡度過陡
- extremely_high: 距離河川 <500m 且坡度過陡
```

#### 2. **暴雨風險評估**
```python
# 雨量閾值
- WARNING: 雨量 > 40mm 或地形高風險
- URGENT: 雨量 > 40mm 且地形高風險
- CRITICAL: 雨量 > 80mm

# 綜合風險
if 雨量 > 80 → CRITICAL
elif 雨量 > 40 and 地形高風險 → URGENT
elif 雨量 > 40 or 地形高風險 → WARNING
else → SAFE
```

## 問題與反思

### 1. 宜蘭縣的資料提取

**📋 挑戰描述**
本次實作需要宜蘭縣的測站以及避難所資料，然而過去 Week 3 和 Week 4 僅有花蓮縣的實作成果，因此本次實作之前必須將宜蘭縣的完整風險分析做好。

**🔄 處理流程**
- **Step 1**: 繪製宜蘭縣 DEM (需要先 reproject、轉成 .tif 檔、再將所有宜蘭縣散圖拼起來)
- **Step 2**: 將宜蘭縣的避難所以及河川裁剪出來
- **Step 3**: 根據 Week 4 的 terrain_risk 幫宜蘭縣的每個避難所 label
- **Step 4**: 最後合併宜蘭縣以及花蓮縣的風險資料，接著才能進行本次的實作部分

**💡 學習心得**
這部分真的花了不少時間，因為我需要找已經篩選過的全臺灣合理點位避難所的 CSV 檔案以節省時間，再加上很多需要匯入的檔案，因此我學到要好好將過去所掌握的檔案及資料整理清楚以加速未來的操作流程。

---

### 2. sjoin 結果為空（CRS 未對齊）問題解決

**📋 挑戰描述**
本次實作中有許多不同的圖層，過去曾發生過 CRS 沒有對齊而產生空值。

**🛠️ 解決方案**
- **預防措施**: 在每一次的 sjoin 等需要對齊 CRS 前都會再次確認投影座標
- **除錯策略**: 此方法不但能減少類似錯誤發生，也能在發現問題時迅速找到問題出在哪一個 cell 而快速修正

**💡 技術收穫**
學會了系統性的 CRS 管理方法，建立了穩健的空間分析工作流程。

### 3. 視覺化地圖製作

**📋 進階技巧**
- **Marker Cluster**: 主動添加 marker cluster 提升地圖辨識能力
- **分層著色**: 將不同性質的圖層用不同顏色處理，提高視覺區分度

## 🔧 技術規格

### 📦 主要套件
- **Python 3.8+**
- **GeoPandas**：地理資料處理
- **Rasterio + RioXarray**：DEM 處理
- **Folium**：互動式地圖
- **Pandas**：資料分析
- **Requests**：API 呼叫

### 📐 座標系統
- **輸入資料**：EPSG:4326 (WGS84)
- **分析處理**：EPSG:3826 (TWD97/TM2 zone 121)
- **輸出地圖**：EPSG:4326 (Web Mercator)

### 🌐 API 規格
- **CWA O-A0002-001**：自動雨量站即時觀測
- **認證方式**：API Key Authorization
- **更新頻率**：即時 (每10分鐘)

## 🚨 使用注意事項

### ⚠️ 資料安全
- **API Key**：請勿公開分享，儲存在 `.env` 檔案
- **個人資料**：避難所地址等敏感資料需注意隱私
- **檔案權限**：確保輸出目錄有寫入權限

### 🔧 系統需求
- **記憶體**：建議 8GB+ (DEM 處理較耗資源)
- **網路**：需要網路連線 (API 呼叫)
- **硬碟空間**：至少 2GB (DEM 資料)

### 🐛 常見問題
1. **GDAL 錯誤**：請安裝 GDAL 完整套件
2. **API 失敗**：檢查網路連線與 API Key
3. **座標轉換**：確認 CRS 設定正確
4. **記憶體不足**：分批處理大型 DEM 檔案
